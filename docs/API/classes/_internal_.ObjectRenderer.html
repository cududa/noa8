    Class ObjectRenderer

    A class that renders objects to the currently bound render target. This class only renders objects, and is not concerned with the output texture or post-processing.

    dontSetTransformationMatrix

    dontSetTransformationMatrix: boolean

    If true, the object renderer will not set the view/projection/transformation matrices for the active camera (default: false). By default, the view/projection/transformation matrices are set from the active camera (either ObjectRenderer.activeCamera or scene.activeCamera). Sets this property to true if you want to define your own transformation matrices (use the onInitRenderingObservable observable to set your own matrices, to be sure they will be correctly taken into account)

    enableBoundingBoxRendering

    enableBoundingBoxRendering: boolean

    Enables the rendering of bounding boxes for meshes (still subject to Mesh.showBoundingBox or scene.forceShowBoundingBoxes). Default is false.

    enableOutlineRendering

    enableOutlineRendering: boolean

    Enables the rendering of outline/overlay for meshes (still subject to Mesh.renderOutline/Mesh.renderOverlay). Default is true.

    forceLayerMaskCheck

    forceLayerMaskCheck: boolean

    Force checking the layerMask property even if a custom list of meshes is provided (ie. if renderList is not undefined)

    getCustomRenderList

    getCustomRenderList: (
        layerOrFace: number,
        renderList: readonly AbstractMesh[],
        renderListLength: number,
    ) => AbstractMesh[]

    Use this function to overload the renderList array at rendering time. Return null to render with the current renderList, else return the list of meshes to use for rendering. For 2DArray, layerOrFace is the index of the layer that is going to be rendered, else it is the faceIndex of the cube (if the RTT is a cube, else layerOrFace=0). The renderList passed to the function is the current render list (the one that will be used if the function returns null). The length of this list is passed through renderListLength: don't use renderList.length directly because the array can hold dummy elements!

    options

    options: Required<ObjectRendererOptions>

    The options used by the object renderer

    particleSystemList

    particleSystemList: IParticleSystem[]

    Define the list of particle systems to render. If not provided, will render all the particle systems of the scene. Note that the particle systems are rendered only if renderParticles is set to true.

    renderAlphaTestMeshes

    renderAlphaTestMeshes: boolean

    Define if alpha test meshes should be rendered (default is true). No effect if renderMeshes is false.

    renderDepthOnlyMeshes

    renderDepthOnlyMeshes: boolean

    Define if depth only meshes should be rendered (default is true). No effect if renderMeshes is false.

    renderListPredicate

    renderListPredicate: (AbstractMesh: AbstractMesh) => boolean

    Use this predicate to dynamically define the list of mesh you want to render. If set, the renderList property will be overwritten.

    renderMeshes

    renderMeshes: boolean

    Define if meshes should be rendered (default is true).

    renderOpaqueMeshes

    renderOpaqueMeshes: boolean

    Define if opaque meshes should be rendered (default is true). No effect if renderMeshes is false.

    renderParticles

    renderParticles: boolean

    Define if particles should be rendered (default is true).

    renderPassId

    renderPassId: number

    Current render pass id. Note it can change over the rendering as there's a separate id for each face of a cube / each layer of an array layer!

    renderSprites

    renderSprites: boolean

    Define if sprites should be rendered (default is false).

    renderTransparentMeshes

    renderTransparentMeshes: boolean

    Define if transparent meshes should be rendered (default is true). No effect if renderMeshes is false.

    Static ReadonlyREFRESHRATE_RENDER_ONCE

    REFRESHRATE_RENDER_ONCE: number

    Objects will only be rendered once which can be useful to improve performance if everything in your render is static for instance.

    Static ReadonlyREFRESHRATE_RENDER_ONEVERYFRAME

    REFRESHRATE_RENDER_ONEVERYFRAME: number

    Objects will be rendered every frame and is recommended for dynamic contents.

    Static ReadonlyREFRESHRATE_RENDER_ONEVERYTWOFRAMES

    REFRESHRATE_RENDER_ONEVERYTWOFRAMES: number

    Objects will be rendered every 2 frames which could be enough if your dynamic objects are not the central point of your effect and can save a lot of performances.
