    Class Text

    noa.text - 3D text rendering subsystem (optional)

    Provides factory methods for creating 3D text meshes in the world. Uses meshwriter-cudu for text-to-mesh conversion with Babylon.js.

    This module is optional - it only initializes if meshwriter is available. If meshwriter is not installed, noa.text.ready will be false and text creation methods will return null.

    This module uses the following default options (from the options object passed to the Engine):

    {
        defaultFont: 'Helvetica',
        scale: 1,
        enabled: true,
    }

    createWorldText

    • createWorldText(content: string, options?: TextOptions): TextHandle

      Create 3D text mesh at a world position. Returns a TextHandle for manipulation and disposal.

      Parameters

      • content: string

        Text string to render

      • Optionaloptions: TextOptions = {}

        Text options

      Returns TextHandle

      • Handle for the text, or null if not ready

      Example

          position: [100, 10, 50],
          letterHeight: 2,
          color: '#8B4513',
      })
      sign.mesh.rotation.y = Math.PI / 4

    loadAndRegisterFont

    • loadAndRegisterFont(name: string, fontFile: string): Promise<void>

      Load and register a built-in font in one step. Convenience method that combines loadFont() and registerFont().

      WARNING: This does NOT work in browser environments! See loadFont() documentation for details.

      For browsers, use static imports:

      import fontData from 'meshwriter-cudu/fonts/atkinson-hyperlegible-next'
      noa.text.registerFont('Atkinson', fontData)

      Parameters

      • name: string

        Name to register the font as

      • fontFile: string

        Built-in font filename (e.g., 'atkinson-hyperlegible-next')

      Returns Promise<void>

    loadFont

    • loadFont(fontFile: string): Promise<any>

      Load a built-in font by filename (without importing meshwriter-cudu directly). Returns a promise that resolves with the font data.

      WARNING: This does NOT work in browser environments! Bundlers (Vite, webpack) cannot resolve dynamic imports with variable paths. For browsers, use static imports instead:

      import fontData from 'meshwriter-cudu/fonts/atkinson-hyperlegible-next'
      noa.text.registerFont('Atkinson', fontData)

      This method is only useful in Node.js environments (server-side rendering, tools).

      Parameters

      • fontFile: string

        Built-in font filename (e.g., 'helvetica', 'atkinson-hyperlegible-next', 'jura')

      Returns Promise<any>

      Font data object

    updateText

    • updateText(handle: TextHandle, newContent: string): TextHandle

      Update text content (dispose old, create new). MeshWriter doesn't support in-place updates, so this recreates the mesh. Position, rotation, and options are preserved.

      Parameters

      • handle: TextHandle

        Existing text handle

      • newContent: string

        New text content

      Returns TextHandle

      • New handle (old is disposed)

      Example

      const newHandle = noa.text.updateText(oldHandle, 'New text content')
