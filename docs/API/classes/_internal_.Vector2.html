    Static_V8PerformanceHack

    _V8PerformanceHack: DeepImmutable<Vector2>

    If the first vector is flagged with integers (as everything is 0,0), V8 stores all of the properties as integers internally because it doesn't know any better yet. If subsequent vectors are created with non-integer values, V8 determines that it would be best to represent these properties as doubles instead of integers, and henceforth it will use floating-point representation for all Vector2 instances that it creates. But the original Vector2 instances are unchanged and has a "deprecated map". If we keep using the Vector2 instances from step 1, it will now be a poison pill which will mess up optimizations in any code it touches.
